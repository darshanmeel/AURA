# 404 / Not-found state

**URL:** `/sessions/<unknown-id>` (or any non-existent dynamic route)  
**Status:** 404

## What this screen shows

Default Next.js 404 page with gray background, "404" heading, and "Not Found" message. Accessible navigation back to the app.

## Trigger

Server component calls `notFound()` from `next/navigation` when the entity lookup returns null. The query fails to find a matching database record.

## Routes that use this pattern

- `/sessions/[sessionId]`
- `/apps/[appId]`
- `/people/[personId]`

## Rendering

No custom `not-found.tsx` component exists; Next.js uses its default error boundary for 404 responses.
