# Aura v0.1 Core Implementation Design

**Goal:** Establish a real-time ingestion pipeline and live dashboard for AI agent transcripts.

## Architecture
1. **Watcher (Python):** Monitors JSONL files, parses Claude events, computes `context_pct`, and writes to `aura.duckdb`.
2. **Snapshot Worker:** Atomic copy of `aura.duckdb` to `aura_read.duckdb` every 2s for concurrent read access.
3. **Streamlit UI:** Reads `aura_read.duckdb` to display live session metrics and today's aggregate usage.

## Components

### 1. Watcher & Claude Adapter
- **Watchdog:** Monitors `~/.claude/projects/**/*.jsonl`.
- **Checkpointing:** Tracks `last_offset` per file in `ingest_checkpoints`.
- **Normalization:** `raw_events` table stores denormalized usage and context data.
- **Redaction:** Optional regex-based scrubbing of secrets from `payload`.

### 2. DuckDB Storage
- **Two-File Pattern:** `aura.duckdb` (exclusive write) -> `aura_read.duckdb` (read-only).
- **Snapshot Logic:** `PRAGMA force_checkpoint` -> `os.replace` for atomicity.

### 3. Live Dashboard (Streamlit)
- **Auto-Refresh:** `st.fragment` every 2s for live session tracking.
- **Fall-back Aggregates:** Direct SQL over `raw_events` for today's totals (pre-dbt).

## Success Criteria
- [ ] New agent turns appear in UI within 4 seconds.
- [ ] No data loss on watcher restart (idempotent offsets).
- [ ] Secrets redacted from `payload` when configured.
